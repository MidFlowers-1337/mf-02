# 自习室工位预约系统

Node.js + Koa + SQLite (better-sqlite3) + 原生 JS 前端

## 快速启动

```bash
npm install
npm start
```

或者双击运行 `启动.bat`（Windows）

启动后访问：
- **学生端**：http://localhost:3000
- **老板端**：http://localhost:3000/admin.html

默认管理员账号：`admin` / `admin123`

## 功能清单

### 1. 工位管理
- 工位属性：编号、区域、是否有屏、是否靠窗
- 状态：可预约 / 维护中
- 老板端可增删改查

### 2. 学生注册登录
- JWT 鉴权
- 密码 bcrypt 加密

### 3. 预约
- 按日期 + 时段（每小时一段，8:00-22:00）+ 工位 预约
- 同一天同一时段一个人只能约一个工位
- 工位被占了不让约
- 不能预约过去的时段
- 维护中的工位不让约

### 4. 签到释放
- 预约开始时间前不能签到
- 超过预约开始时间 15 分钟没签到 → 自动释放工位 + 记一次爽约
- 定时任务每 60 秒轮询一次释放超时预约

### 5. 结束使用
- 用完点「结束使用」释放工位

### 6. 爽约禁约
- 7 天内爽约 3 次 → 禁约 7 天
- 禁约期间不能预约新工位
- 7 天后自动解除

### 7. 老板后台
- 工位利用率统计（按日期范围）
- 热门时段排名
- 区域使用分布
- 用户列表（含禁约状态）
- 爽约记录
- 预约记录
- 工位管理

## 验证步骤

### 1. 预约工位
1. 打开 http://localhost:3000
2. 注册一个账号并登录
3. 选择日期 → 选择时段 → 点击一个空闲工位 → 点「确认预约」
4. 在「我的预约」列表中可以看到刚约的工位，状态是「已预约」

### 2. 签到使用
1. 预约时段开始后，点「签到」
2. 状态变成「使用中」
3. 用完点「结束使用」，状态变成「已完成」

### 3. 模拟超时释放
1. 预约一个**当前小时**的工位（比如现在 14:30，约 14:00 的）
2. **不要**签到
3. 等待最多 60 秒（定时任务轮询间隔）
4. 查看服务器控制台，应该看到 `[scheduler] 已释放 1 个超时预约`
5. 刷新页面，预约状态变成「爽约」，并记录一次爽约

### 4. 爽约禁约
1. 重复步骤 3 三次（故意爽约 3 次）
2. 第 4 次预约时，会提示「您因爽约3次已被禁约，剩余 168 小时解除」
3. 页面顶部会出现红色禁约提示

### 5. 时段冲突测试
1. 用账号 A 预约 A01 工位 10:00
2. 用账号 B 同时段也约 A01，应该提示「该工位此时段已被预约」

### 6. 同时段多工位测试
1. 同一账号约 A01 工位 10:00
2. 再约 A02 工位 10:00，应该提示「您同时段已预约了其他工位」

### 7. 老板端看统计
1. 打开 http://localhost:3000/admin.html
2. 登录 admin / admin123
3. 可以看到：
   - 工位利用率百分比
   - 热门时段柱状图
   - 区域使用占比
   - 用户列表（含禁约状态、爽约次数）
   - 爽约记录
   - 预约记录

## 核心模块说明

### src/booking.js
所有业务判断都集中在这里，路由层只做参数透传：
- `createBooking()` - 创建预约（含所有冲突检查）
- `checkin()` - 签到（含时间检查）
- `checkout()` - 结束使用
- `markNoShow()` - 标记爽约
- `releaseExpiredBookings()` - 批量释放超时预约（定时任务调用）
- `getBanInfo()` - 获取用户禁约状态
- `canBookSeat()` / `canUserBookSlot()` - 冲突检查
- `getUtilizationStats()` - 利用率统计

### src/scheduler.js
每 60 秒轮询一次，调用 `releaseExpiredBookings()` 释放所有超时未签到的预约。

## 技术选型理由

- **Koa**：轻量中间件机制，符合要求
- **better-sqlite3**：同步 API，不用回调，性能好，符合要求
- **原生 JS 前端**：零构建，打开就能用，符合要求
- **JWT**：无状态鉴权，前端存 localStorage
- **bcryptjs**：密码加密
- 不引 ORM、不引队列、不引前端框架，全部手写
